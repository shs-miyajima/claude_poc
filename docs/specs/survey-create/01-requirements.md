# 要件定義 — アンケート作成（設問作成・編集・削除・下書き/公開・代理作成）

> status: 01-requirements.status を参照（`draft` | `approved` | `rejected`）
>
> `open-questions.md` の 11 件（#1〜#11）は全件 resolved。回答内容はすべて本文に反映済み。

## 1. 概要

管理者（企業側の担当者）およびスーパーユーザー（個別企業画面に切り替えた状態での代理作成）が、
単一選択・複数選択・自由記述・段階評価の 4 種類の設問を持つアンケートを作成・編集・削除する機能。
アンケートは下書き / 公開の 2 状態を持ち、下書きの間のみアンケート本体・設問・選択肢の
追加・編集・削除が可能。公開すると以降の編集・削除は不可となり、設定した回答期間（開始日・終了日）と
記名/匿名の区分に従って、ユーザー（`survey-answer` スペックで実装する回答機能）からの
回答対象になる。公開されたアンケートは対象企業の全ユーザーに配布され、部署・属性等による配布対象の
選択機能はない（親決定 Q16）。

## 2. 用語集

| 用語 | 定義 |
|------|------|
| 下書き（draft） | アンケートの初期状態。アンケート本体・設問・選択肢の追加・編集・削除が可能。ユーザーには配布されない |
| 公開（published） | 下書きから公開操作をした後の状態。以降アンケート本体・設問・選択肢の編集・削除は不可（親決定 Q18）。回答期間内であればユーザーが回答可能になる（回答自体は `survey-answer` スペックの対象） |
| 代理作成 | スーパーユーザーが個別企業画面（`survey-accounts` の画面切替機能）に切り替えた状態で、その企業の管理者と同じアンケート作成画面を使ってアンケートを作成すること（親決定 Q19）。作成者情報は画面に表示しない（open-questions #7） |
| 記名 / 匿名 | アンケート作成時に設定する回答閲覧方式の区分。匿名は「画面仕様レベルの匿名」（親決定 Q30）。DB では回答とユーザーの紐づけを保持するが、匿名アンケートでは管理者向け画面に個人を特定できる情報（回答者名・個票との紐づけ）を表示しない |
| 段階評価 | 1〜5 の 5 段階固定のスケール形式の設問（親決定 Q34）。選択肢マスタを持たず、回答は 1〜5 の数値のみ。両端（1 と 5）に任意でラベルを設定できる（open-questions #2） |

## 3. スコープ

### 対象

- [x] Laravel 管理画面
- [x] フロント（Blade / Vite / JavaScript / Tailwind CSS）
- [x] DB スキーマ変更（アンケート・設問・選択肢テーブルの新設）

### 対象外

- ユーザーによるアンケートの回答・下書き回答保存・回答期間内修正・回答済み確認
  （`survey-answer` スペックで対応）
- ダッシュボード（属性による絞り込み・グラフ表示・回答率の集計）（`survey-dashboard` スペックで対応）。
  本機能の一覧・詳細画面では回答数・回答率を表示しない（open-questions #11）
- 設問の CSV 一括登録・インポート（仕様に記載がなく、今回はスコープに含めない）
- 公開後のアンケート・設問・選択肢の編集・削除（親決定 Q18 で「下書きのみ可」と確定済み。
  本機能でこの制約を変更しない）
- 複数選択設問の選択可能数（最小/最大）の制約設定（open-questions #10 で「不要」と確定。
  常に 1 個以上・上限なしで選択可能）
- アンケートの配布対象の選択（部署・属性等によるターゲティング）（親決定 Q16 で「全ユーザー」と
  確定済み。対象ユーザーを選ぶ機能は設けない）
- 企業・部署・管理者・ユーザーの登録・編集・削除、認証、全体/個別企業の画面切替の仕組み自体
  （`survey-accounts` スペックで実装済み。本機能はその基盤（ログイン・ロール・画面切替）の上に構築する）

## 4. ユーザーストーリー / ユースケース

| ID | 操作者 | 操作 | 期待結果 |
|----|--------|------|----------|
| UC-01 | 管理者 | アンケートを新規作成し下書き保存する（タイトル・回答期間・記名/匿名を入力） | アンケートが下書き状態で保存され、一覧に表示される |
| UC-02 | 管理者 | 下書きのアンケートにタイトル・回答期間・記名/匿名の変更を加えて保存する | 変更内容が保存される |
| UC-03 | 管理者 | 下書きのアンケートを削除する | アンケート（配下の設問・選択肢を含む）が物理削除され、一覧から消える |
| UC-04 | 管理者 | 下書きのアンケートを公開する | アンケートが公開状態になり、以後編集・削除が不可になる |
| UC-05 | 管理者 | アンケート一覧を閲覧する（下書き・公開の両方） | 自社のアンケートが状態表示付きで一覧表示される（20 件/ページ） |
| UC-06 | 管理者 | アンケートに設問を追加する（単一選択・複数選択・自由記述・段階評価のいずれか） | 設問が追加され、アンケートに紐づく。必須/任意は初期値「任意」 |
| UC-07 | 管理者 | 下書き中の設問を編集する（設問文・形式・必須/任意・選択肢等） | 変更内容が保存される |
| UC-08 | 管理者 | 下書き中の設問を削除する | 設問（配下の選択肢を含む）が物理削除される |
| UC-09 | 管理者 | 下書き中の設問の並び順を変更する | 変更後の並び順で保存される |
| UC-10 | 管理者 | 単一選択・複数選択設問の選択肢を追加・編集・削除する（2〜10 件） | 選択肢の変更が保存される |
| UC-11 | 管理者 | 段階評価設問の両端ラベルを設定する（任意入力） | ラベルが保存される。未入力の場合は数値のみ表示される |
| UC-12 | スーパーユーザー | 個別企業画面に切り替えた状態でアンケートを代理作成する | UC-01〜UC-10 と同じ操作が、切替中の企業のアンケートとして実行される。作成者情報は画面に表示されない |
| UC-13 | 管理者 | アンケート詳細画面で設問構成（設問文・形式・選択肢・必須/任意）を確認する | 保存済みの構成が表示される |

## 5. 画面・API 仕様

### 5.1 画面

| 画面名 | URL（想定） | 遷移元 | 備考 |
|--------|-------------|--------|------|
| アンケート一覧 | `/company/surveys` | 企業ホーム画面のメニュー | 管理者・スーパーユーザー（個別企業画面）共通。20 件/ページ |
| アンケート作成 | `/company/surveys/create` | アンケート一覧の「新規作成」 | 設問・選択肢を同一画面内で組み立てる想定（詳細は 02-design で確定） |
| アンケート編集 | `/company/surveys/{survey}/edit` | アンケート一覧・詳細の「編集」（下書きのみ表示） | 公開済みアンケートへの直接アクセスは 403（§7 VAL-14） |
| アンケート詳細 | `/company/surveys/{survey}` | アンケート一覧の行クリック | 下書き・公開の両方で閲覧可能。作成者情報は表示しない |

### 5.2 画面項目定義

| 項目名 | 型 | 必須 | 最大長 | 初期値 | 表示条件・備考 |
|--------|----|------|--------|--------|----------------|
| タイトル | 文字列 | 必須 | 500 文字（親決定 Q33） | なし | 同一企業内での重複を許容（一意制約なし。open-questions #9） |
| 回答期間（開始日） | 日付 | 必須（親決定 Q35） | — | なし | 終了日以前の日付のみ許可（詳細は §7） |
| 回答期間（終了日） | 日付 | 必須（親決定 Q35） | — | なし | 開始日以降の日付のみ許可 |
| 記名/匿名 | 選択（記名 / 匿名） | 必須 | — | なし（未選択で保存不可） | 親決定 Q22, Q30 |
| 設問文 | 文字列 | 必須 | 500 文字（親決定 Q33） | なし | — |
| 設問形式 | 選択（ラジオボタン。単一選択 / 複数選択 / 自由記述 / 段階評価） | 必須 | — | 単一選択 | 親決定 Q13, Q34。新規追加した設問ブロックは「単一選択」があらかじめ選択された状態で表示する（実装完了後のユーザーレビューにより確定）。URL直接アクセス等でquestion_typeが空欄のまま送信された場合はVAL-16で拒否する |
| 設問の必須/任意 | 選択（ラジオボタン。必須 / 任意） | 必須 | — | 任意（open-questions #3） | 親決定 Q17 で設定可能自体は確定済み。作成者が必要な設問のみ意図的に必須へ切り替える運用 |
| 選択肢文言（単一選択・複数選択） | 文字列 | 必須 | 500 文字（親決定 Q33） | なし | 1 設問あたり 2〜10 件（下限: open-questions #4、上限: 親決定 Q14）。個々の選択肢の未入力は VAL-17 |
| 段階評価の両端ラベル（1 側 / 5 側） | 文字列 | 任意 | 500 文字（親決定 Q33 の一律 500 に合わせる） | なし | 段階評価設問で常に入力欄を表示。未入力なら詳細・回答画面では数値のみ表示（open-questions #2） |

### 5.3 API / ルート

<!-- 詳細なルート定義は 02-design.md で確定する。フェーズ1では操作単位の概要のみ -->

- アンケート一覧取得、新規作成（下書き保存）、編集（下書きのみ）、削除（下書きのみ・物理削除）、公開
- 設問・選択肢の追加・編集・削除は、アンケート作成/編集画面内の操作としてアンケートの保存に
  含める想定（詳細は 02-design.md で確定。個別 API に分けるか一括保存にするかは設計判断）

### 5.4 フロント連携

- 作成/編集画面での設問の動的追加・削除（JavaScript）
- 単一選択・複数選択設問の選択肢の動的追加・削除（JavaScript）
- 設問の並び順変更 UI（JavaScript。ドラッグ&ドロップ等の具体方式は 02-design で確定）

## 6. データ要件

| テーブル | 操作 | 概要 |
|----------|------|------|
| surveys（新設） | CRUD（削除は物理削除） | アンケート本体。企業・作成者・タイトル・回答期間・記名/匿名・状態（下書き/公開）を保持 |
| questions（新設） | CRUD（削除は物理削除） | 設問。アンケートに紐づく。設問文・形式・必須/任意・並び順・（段階評価の場合）両端ラベルを保持 |
| choices（新設） | CRUD（削除は物理削除） | 選択肢。単一選択・複数選択の設問に紐づく。選択肢文言・並び順を保持 |

削除方式は物理削除（open-questions #6）。下書きは公開前で回答実績が存在しないため、
`survey-accounts` の企業・アカウント等（論理削除方針・親決定 Q32）とは異なる方針とする。

## 8. 権限

| ロール | 操作可否 |
|--------|----------|
| スーパーユーザー（全体画面） | アンケートの作成・編集・削除・閲覧は不可（個別企業画面への切替が必要。親決定 Q19） |
| スーパーユーザー（個別企業画面） | 切替中企業のアンケートに対し、管理者と同等の作成・編集（下書きのみ）・削除（下書きのみ）・公開・閲覧が可能。作成者情報は画面に表示されない（open-questions #7） |
| 管理者 | 自社のアンケートに対し、作成・編集（下書きのみ）・削除（下書きのみ）・公開・閲覧が可能。他社のアンケートは不可 |
| ユーザー | 作成・編集・削除・公開・閲覧（本機能の管理画面）は不可（本機能の対象外。回答は `survey-answer` スペックの対象） |

## 9. 非機能要件

| ID | 非機能要件 | 備考 |
|----|-----------|------|
| NFR-01 | 想定規模（企業数・アンケート数・設問数等）に基づく性能目標は定義しない | 親決定 Q27 でシステム全体の性能目標定義をスコープ外とした方針を継承 |
| NFR-02 | テナント分離: 管理者・個別企業画面のスーパーユーザーは自社（切替中企業）以外のアンケートにアクセスできない | 親決定 Q19, Q26, Q29 を継承。VAL-15 で検証 |
| NFR-03 | ロール外・他社の URL への直接アクセスは 403 で拒否する（画面にリンクを出さないだけの制御にしない） | `survey-accounts` の NFR-03 と同方針を継承。VAL-14, VAL-15 で検証 |

## 10. 具体例（入出力サンプル）

### 正常系の例

- タイトル「2026年度 従業員満足度調査」、回答期間 2026-08-01〜2026-08-31、記名/匿名「匿名」で
  アンケートを新規作成し、単一選択設問「あなたの部署は？」（必須、選択肢「営業部」「開発部」「総務部」の
  3 件）を追加して下書き保存する → アンケートが「下書き」状態で一覧に表示され、詳細画面で
  設問文・選択肢 3 件・必須が表示される
- 同一企業内で既に登録済みの「2026年度 従業員満足度調査」と同じタイトルで別のアンケートを
  新規作成して保存する → タイトルの重複によるエラーにならず、両方が一覧に表示される（open-questions #9）
- 選択肢をちょうど 10 件登録して保存する → 成功する（境界値・上限ちょうど）。11 件目を追加しようとすると
  「選択肢は 10 件まで登録できます」のエラーが表示され追加されない（VAL-09）
- 選択肢をちょうど 2 件登録して保存する → 成功する（境界値・下限ちょうど）。1 件のみで保存しようとすると
  「選択肢は 2 件以上登録してください」のエラーが表示され保存されない（VAL-10）
- 段階評価設問を、両端ラベルを入力せずに保存する → 詳細画面ではラベルなし（数値 1〜5 のみ）で表示される。
  1 側に「全くそう思わない」、5 側に「とてもそう思う」を入力して保存すると、詳細画面に入力した
  ラベルが表示される
- タイトル・設問文・選択肢文言・段階評価ラベルをそれぞれちょうど 500 文字で入力して保存する →
  いずれも成功する（境界値・上限ちょうど。501 文字ならそれぞれ VAL-02, VAL-07, VAL-11, VAL-13 でエラー）
- アンケートにちょうど 100 件の設問を登録して保存する → 成功する（境界値・上限ちょうど。101 件目を
  追加しようとすると「設問は 100 件まで登録できます」のエラーが表示され追加されない。VAL-08）

### 異常系の例

- 回答期間の開始日に 2026-08-01、終了日に 2026-07-31（開始日より前）を入力して保存する →
  「回答終了日は開始日以降の日付を入力してください」のエラーが表示され保存されない（VAL-04）
- 下書きのアンケートを公開した後、そのアンケートの編集画面 URL（`/company/surveys/{id}/edit`）に
  直接アクセスする → 403 になり編集できない（VAL-14）
- 管理者が自社以外のアンケートの詳細画面 URL に直接アクセスする → 403 になり閲覧できない（VAL-15）

| AC-09 | 単一選択・複数選択設問で選択肢を 11 件目追加しようとするとエラーになり追加されない |
| AC-10 | アンケートあたりの設問を 101 件目追加しようとするとエラーになり追加されない |
| AC-11 | 単一選択・複数選択設問で選択肢が 1 件のみの状態で保存しようとするとエラーになり保存されない |
| AC-12 | 作成/編集画面で設問の並び順を変更して保存すると、変更後の順序で詳細画面に表示される |
| AC-13 | 段階評価設問の両端ラベルを入力せずに保存すると、詳細画面で数値のみ（ラベルなし）で表示される |
| AC-14 | 段階評価設問の両端ラベルを入力して保存すると、詳細画面に入力したラベルが表示される |
| AC-15 | 既存アンケートと同じタイトルで新しいアンケートを作成して保存すると、エラーにならず両方が一覧に表示される |
| AC-16 | スーパーユーザーが代理作成したアンケートの一覧・詳細画面に、作成者を特定できる情報が表示されない |
| AC-17 | 設問形式を選択せずに設問を保存しようとするとエラーになり保存されない |
| AC-18 | 単一選択・複数選択設問で、いずれかの選択肢の文言が未入力のまま保存しようとするとエラーになり保存されない |
