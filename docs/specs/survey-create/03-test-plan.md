### 1.2 E2E に載せないもの（PHPUnit / Vitest へ）

- 境界値の NG/OK ペア（タイトル・設問文・選択肢文言・段階評価ラベルの 500/501 文字、設問数 100/101 件、選択肢数 2/1 件・10/11 件）はすべて PHPUnit に配置（§1.3・§1.4）
- 複合 VAL（VAL-03・VAL-09・VAL-10・VAL-13・VAL-17）のサブルールごとの網羅は PHPUnit に配置
- ロール×URL の 403 網羅（VAL-14・VAL-15・§8 権限）は PHPUnit HTTP に配置し、E2E では扱わない（画面固有の 403 表示要素がないため）
- 設問・選択肢の追加/削除/並び替えの DOM 操作ロジック・name 属性再採番・設問形式による入力欄表示切替は Vitest に配置し、E2E は保存結果の反映確認（代表 1 ジャーニー）のみ
- 段階評価ラベル有無による詳細表示の出し分けは PHPUnit Feature（assertSee 相当）で確認し、E2E では扱わない
- タイトル重複許容の確認は PHPUnit で十分（一意制約なしのロジック確認）

### 期待結果の書き方（曖昧表現の排除）

全 CSV の期待結果は、メッセージ文言・件数・画面状態など検証可能な具体値で記載する（「正しく」「適切に」「問題なく」等は使用しない）。

### 独立性・再現性の標準観点

- 各 PHPUnit ケースは `RefreshDatabase` を用いた Feature/Unit テストとして、ケースごとに前提データ（企業・管理者・アンケート・設問・選択肢）を `SurveyFactory` / `QuestionFactory` / `ChoiceFactory` 等でテスト内に新規作成する。他ケースが作成したレコードを暗黙の前提にしない
- タイトル重複（PU-058）のような同一値を扱うケースも、テストごとに専用の企業・アンケートを新規作成するため他ケースと衝突しない
- E2E は `migrate:fresh --seed` 直後の状態を前提とし、各ケース内で `tests/e2e_tests/tests/helpers.ts` の `loginAsSuperUser` / `registerCompany` / `switchToCompany` / `registerAdmin` を用いて専用の企業・管理者を新規作成してからアンケート操作を行う（他ケースが作成した企業・アンケートを更新・参照しない）
- PU-052／E2E-007（ページング）のような 21 件データを前提とするケースは、当該ケース専用の企業を作成しその配下にアンケートを作成することで、他ケースの一覧件数に影響しない

#### 並列実行安全性（E2E は並列実行が前提）

- E2E の各ケースは、操作対象の企業・管理者・アンケートをケースごとに新規作成してから操作する（Seeder 由来の共有データを直接更新しない）
- E2E-004（削除）・E2E-005（公開）は当該テスト内で作成した専用アンケートを対象とし、他ケースが参照するアンケートには影響しない
- E2E-007（ページング）は当該テスト専用に 21 件のアンケートを作成し、他ケースが作成するアンケート数に依存しない

### CSV ケース粒度の標準観点

各 CSV は 1 行 1 観点。複合 VAL のサブルールごと、設問形式（単一選択/複数選択）ごと、新規/更新ごと、ロール・権限パターンごとに個別行として作成済み（§1.3・§1.4 参照）。

### 更新系の標準観点（編集機能がある場合は必須）

対象: アンケート本体（タイトル・回答期間・記名/匿名）・設問（設問文・設問形式・選択肢文言）。段階評価の両端ラベルのみ任意項目。
