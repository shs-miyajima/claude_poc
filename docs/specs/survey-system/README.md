# アンケート作成・回答・管理システム — 全体概要（親スペック）

> 本フォルダは分割スペックの**親**であり、全体概要とシステム全体の Q&A（`open-questions.md`）を
> 保持する。承認ゲート（`*.status`）は持たない。各機能の要件・設計・テスト計画は
> 分割スペック側（下表）が正本。

## 1. システム概要

企業向けのアンケート作成・回答・管理システム。
管理者（企業の人事部等）がアンケートを作成し、ユーザー（企業の社員）が回答し、
回答結果をダッシュボードでグラフ形式（縦棒グラフ / 円グラフ）で確認できる。
弊社社員はスーパーユーザーとして企業・アカウントの登録と全企業の横断確認・代理操作を行う。

## 2. ロール

| ロール | 対象 | 概要 |
|--------|------|------|
| スーパーユーザー | 弊社社員 | 企業・部署・管理者・ユーザーの管理、全体/個別企業の画面切替、代理アンケート作成 |
| 管理者 | 企業の人事部など一部の担当者 | 自社のユーザー・部署管理、アンケート作成、ダッシュボード閲覧 |
| ユーザー | 企業の社員 | 配布されたアンケートへの回答（下書き・期間内修正・回答済確認） |

## 4. システム全体の決定事項（`open-questions.md` の回答より）

分割スペックをまたぐ決定のサマリ。個々の根拠・経緯は `open-questions.md`（Q1〜Q41）を参照。

| 分類 | 決定 | 出典 |
|------|------|------|
| 認証 | 共通 1 ログイン画面。企業コード + メールアドレス + パスワード（スーパーユーザーは企業コード空欄）。メールアドレスは企業単位で一意 | Q2, Q3, Q7, Q31 |
| 認証 | パスワードは最小 8 文字のみ。ロックアウト・複雑さ要件・変更強制なし。初期パスワードは登録者が入力。初期スーパーユーザーはシーダー投入 | Q4, Q5, Q6 |
| テナント分離 | 管理者・ユーザーは自社データのみアクセス可。スーパーユーザーは全企業にアクセス可（個別企業画面 = 当該企業の管理者画面と同等） | Q19, Q26, Q29 |
| アカウント | 削除はすべて論理削除。削除ユーザーの回答は集計に残す。企業削除時は配下も一括無効化（データ保持） | Q9, Q32 |
| ユーザー属性 | 生年月日・入社年月日・性別・部署。部署は企業ごとのマスタ管理 | Q11, Q36 |
| アンケート | 設問 4 形式（単一選択・複数選択・自由記述・段階評価〔1〜5 固定〕）。設問 100・選択肢 10・作成側文字数一律 500。回答期間（開始/終了）必須。下書き/公開の 2 状態、下書きのみ編集・削除可。配布は全ユーザー。設問ごとに必須/任意設定可 | Q13〜Q18, Q33, Q34, Q35 |
| 匿名 | 作成時に記名/匿名を選択。匿名は「画面仕様レベルの匿名」（DB は紐づけ保持・管理者向け画面に個人特定情報を出さない・回答画面に注記表示） | Q22, Q30 |
| 回答 | 管理者は回答しない（回答はユーザーのみ）。下書き保存あり。回答期間内は提出後も修正可 | Q8, Q20, Q21 |
| ダッシュボード | 設問ごとに 1 グラフ（縦棒/円切替、複数選択・段階評価もグラフ対象、自由記述はグラフなし）。絞り込みは属性連動（日付は From〜To 範囲指定）。回答率表示あり。全体画面は企業一覧 + 導線のみ | Q13, Q23〜Q26, Q37 |
| 環境 | composer の追加パッケージなし（認証は Laravel 標準セッション認証を自前実装）。グラフは npm の Chart.js を追加 | Q28 |
| スコープ外 | 想定規模（性能目標）の定義はスキップ。メール送信・パスワードリセットは対象外 | Q5, Q27 |
