## 3. 削減効果

**フェーズ 4 完了時に算出する。**

フェーズ2の実績時間が複数セッションにまたがり正確な計測不可のため、判明しているフェーズ1・3・4の
3フェーズ分（人手想定 6h+7h+20h=33h）で比較する。フェーズ2を含む人手想定工数の合計（40h）は
参考値として併記する。

| 指標 | 値 |
|------|-----|
| 人手想定工数（フェーズ1・3・4 合計） | 33h |
| 実績時間（フェーズ1・3・4 合計） | 2h39m（約 2.65h） |
| 削減時間 | 約 30.35h |
| 削減率 | 約 92%（(33 − 2.65) / 33） |
| 人手想定工数（フェーズ1〜4 全体・参考） | 40h（約 5 人日） |

### 所感（任意）

- フェーズ1・3・4は着手/承認（完了）のチャット時刻が明確に記録できており、削減率は約 92% と
  非常に高い。ただし工数見積もりは「AI レビュー待ち等の人間の確認時間を含む実時間」であり、
  実装量（新規ファイル約 40 件・PHPUnit 136 件・E2E 17 件）に対して 1h14m で完了できたのは、
  仕様・設計・テスト計画が各フェーズで十分に具体化されていたことが大きい。
- フェーズ4で唯一発見された不具合（企業コード採番の同時実行レースコンディション）は、
  Playwright E2E を並列実行して初めて顕在化した。PHPUnit の単体・結合テストだけでは検出できない
  種類の不具合であり、E2E の並列実行安全性の観点（`03-test-plan.md`）が実際に効果を発揮した例。
