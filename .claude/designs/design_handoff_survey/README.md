# ハンドオフ: アンケート作成・回答・集計システム（権限別UI）

## 概要
社内アンケートの **作成 → 配布 → 回答 → 集計** を、3つの権限（スーパーユーザー / 管理者 / ユーザー）で扱うWeb画面群のUIプロトタイプです。既存の Laravel 12 + Blade + Tailwind のアカウント基盤（ログイン・企業/部署/管理者/ユーザー管理）の上に、**アンケート機能の画面**を追加実装することを想定しています。

## このバンドルのファイルについて
本バンドルの `prototype_source.dc.html` は **HTMLで作られたデザインリファレンス（プロトタイプ）** です。見た目と挙動の意図を示すもので、そのまま本番コードとして貼り付けるものではありません。**このリポジトリの既存環境（Laravel + Blade + Tailwind CSS v4）の流儀で、同じ画面・同じ挙動を再現実装する**のがゴールです。

- ロジック（設問4形式、下書き/公開、回答率算出、属性絞り込み、回答状態遷移など）はプロトタイプの JavaScript クラスに集約されています。ここを**仕様書として読み**、Bladeビュー + Controller + Eloquent へ移植してください。
- プロトタイプのReact的な記述（`renderVals()`、`{{ }}` バインディング、`sc-for`/`sc-if`）は直接は流用不可です。構造とデータ形状の参照元として使ってください。
- 開くには `prototype_source.dc.html` をブラウザで表示（同階層の `support.js` に依存）。上部の「ロール切替」で3権限を、各画面右上のトグルでA/B案を確認できます。

## フィデリティ
**ハイファイ（hifi）**。配色・タイポグラフィ・余白・角丸・状態変化まで作り込んであります。下記「デザイントークン」の値どおりに再現してください。ただしプロトタイプはブランド色をCSS変数化しているため、Blade実装では下記の**確定値（既定のインディゴ）**を採用してください（Tweaksの色替え機能は実装不要）。

## 権限と画面マップ

### スーパーユーザー
- **企業一覧** — 全企業の一覧（企業コード / 企業名 / ユーザー数 / アンケート数 / 状態）。「個別企業画面へ」で特定企業の**代理操作モード**に入る。
- 代理操作中は上部に黄色の「代理操作中」バナーを表示し、管理者と同じ企業内画面（ホーム/アンケート/集計/ユーザー/部署）に加え、**管理者管理**画面を操作できる。「全体画面へ戻る」で解除。

### 管理者（企業内）
- **ホーム** — 指標カード（公開中/下書き/対象ユーザー/平均回答率）＋主要導線2枚。
- **アンケート一覧** — カード形式。状態バッジ（公開中/下書き/終了）、匿名バッジ、回答率バー、「集計を見る」「編集」。
- **アンケート作成/編集** — 後述（A/B 2案）。
- **集計ダッシュボード** — 後述（A/B 2案）。**必ず特定のアンケートに紐づく**（ヘッダーの「対象アンケート」セレクタで選択、または一覧の「集計を見る」から遷移）。
- **ユーザー** / **部署マスタ** — 既存アカウント基盤の一覧（氏名/メール/部署/状態）。

### ユーザー
- **配布アンケート一覧** — 自分に配布されたアンケート。状態（未回答/下書き保存済み/回答済み/期限切れ）、匿名バッジ、残り日数、状態別アクションボタン。
- **回答画面** — 後述（A/B 2案）。回答期間内は提出後も修正可能。匿名アンケートは冒頭に注記。

## 画面詳細

### アプリシェル（全ロール共通）
- 最上部: ダークな「プロトタイプ / ロール切替」ストリップ（**プロトタイプ専用。本番実装では不要**。ロールは認証ユーザーの権限で決定）。
- 左サイドバー（幅 248px、白、右境界 `#e6e8ec`）: ロゴ（グラデーション角丸ブロック "S" + "Survey Console / 社内アンケート基盤"）、権限別ナビ、最下部にユーザー識別（アバター丸 + 氏名 + 役割）。
- ナビ項目のアクティブ: 背景 `#eef0fe`、文字 `#4b53e0`、`border-radius:9px`、`padding:9px 12px`、`font-size:13.5px`。非アクティブ文字 `#4b5563`。
- メイン: `overflow:auto`。ページ見出し h1 は `font-size:22px; font-weight:700`、サブ説明 `#5b6472 / 13.5px`。

### アンケート作成 — 案A「フォーム型」（推奨・既定）
- 中央 `max-width:760px`。
- **メタカード**: 上部に4pxのアクセント上ボーダー。タイトル入力（下線のみ、20px/700）、回答開始日/終了日（`<input type=date>`）、匿名トグル（スイッチ）。
- **設問カード**（設問ごと）: 種別バッジ（`◉単一選択 / ☑複数選択 / ¶自由記述 / ★段階評価`）、設問文入力、種別ごとの本体、右下に「必須」トグル、右上に ▲▼（並べ替え）と削除。
  - 単一/複数選択: 選択肢リスト（丸/角のマーカー）＋「＋選択肢を追加」＋各行×削除。
  - 段階評価: ★×5 と「1〜5の段階評価」注記。
  - 自由記述: 破線プレースホルダ「自由記述の回答欄（最大500文字）」。
- **設問追加**: 破線ボタン → 展開する4種のタイプ選択（グリッド）。

### アンケート作成 — 案B「3ペイン型」
- 左（270px）: 設問アウトライン一覧（番号バッジ＋設問文＋種別）、選択中はハイライト。下に設問追加。
- 中央: 選択中設問のエディタ（種別バッジ、並べ替え/削除、設問文、選択肢、必須トグル）。
- 右（340px）: 回答者ライブプレビュー（白カード、影付き）。

> A/Bどちらか一方を採用でも、両方を切替可能にしてもよい。既定は**案A**を推奨。

### 集計ダッシュボード — 共通
- ヘッダーに **「対象アンケート」セレクタ**（必須。集計は常に単一アンケートにスコープ）。
- レイアウト切替（案A/B）、グラフ切替（**縦棒 / 円**）。
- 指標: 回答率 / 回答者数 / 対象者数 / 設問数（選択アンケートの値に連動）。
- 属性絞り込み: **部署 / 性別 / 年代（入社時期）**。絞り込むとグラフ連動（プロトタイプではUIのみ）。
- 設問種別ごとの集計表示:
  - 単一/複数/段階評価: 縦棒 or 円グラフ（ラベル・件数・割合%）。段階評価は平均値も表示。
  - 自由記述: 回答サンプルをカード列で表示。
- **案A**: 指標カード＋2カラムのグラフカード。**案B**: 左に絞り込みパネル、右にアクセントのKPIヘッダー＋縦積みの大きめグラフ。

### 回答画面 — 案A「1問ずつ（ウィザード）」
- 上部プログレスバー＋「n / 総数」。1問ずつカード表示 → 最後に**確認画面**（全回答サマリ）→ 送信。
- 単一=ラジオ風大ボタン、複数=チェック風、段階評価=1〜5の大ボタン、自由記述=テキストエリア。
- 「戻る」「下書き保存」「次へ／回答を送信」。

### 回答画面 — 案B「一覧回答」
- 左（210px）に回答状況サマリ（回答済み数、設問ごとの✓ドット、sticky）。
- 右に全設問を縦に並べて一括回答。最下部に「下書き保存」「回答を送信」と修正可能の注記。

## インタラクション・挙動
- **ロール/画面遷移**: サイドバーナビ、カード内ボタンで `screen` を切替。
- **代理操作**: 企業一覧「個別企業画面へ」で対象企業をセットし `home` へ。バナーの「全体画面へ戻る」で解除。
- **作成**: 設問の追加（4種）/削除/並べ替え（up/down入替）/フォーカス（案B）、選択肢の追加/削除/編集、設問文・タイトル・期間の編集、必須トグル、匿名トグル、「下書き保存」「公開する」でステータス変更＋トースト表示。
- **集計**: 対象アンケート選択で全指標・グラフが連動。グラフ種別（棒/円）切替。
- **回答**: 単一選択（排他）、複数選択（トグル）、段階評価（単一値）、自由記述（テキスト）。ウィザードは次へ/戻る/確認/送信。下書き保存・送信でステータス変更＋トースト。回答済みでも期間内は再編集可。
- **トースト**: 画面下中央、`#14161c` 背景、約2.2秒で自動消滅。
- アニメーション: `fadeUp`（0.15〜0.2s ease）でカード出現、トーストは `toastIn`。プログレスバーは `width .2s`。

## 状態（State）と主なデータ形状
プロトタイプの `state` を参照。移植時のモデル/テーブルの目安:

- **Company**: `code`(企業コード, 例 C0001), `name`, `status`(active/inactive), `users`(数), `surveys`(数)
- **Admin**: `name`, `email`, 企業に属する
- **User**: `name`, `email`, `department`, 性別・年代・入社時期（絞り込み用属性）
- **Department**: `name`, `status`
- **Survey**: `id`, `title`, `status`(下書き/公開/終了), `start`, `end`(回答期間), `target`(対象者数), `responded`(回答者数), `anonymous`(bool), 設問群
- **Question**: `id`, `type`(`single`|`multi`|`text`|`rating`), `title`, `required`(bool), `options[]`
- **Option**: `id`, `text`
- **Answer/Response**: ユーザー×設問の回答。単一=optionId、複数=optionId配列、段階=1〜5、自由記述=文字列。回答状態（未回答/下書き/回答済/期限切れ）とユーザーの紐付け（匿名時は個人非特定）。
- 回答率 = `responded / target`（%は四捨五入）。段階評価の平均 = Σ(値×件数)/総件数。

## 既存コードとの対応（Laravel）
- 既存の `routes/web.php`・`resources/views/{auth,super,company,user}` のアカウント基盤に合わせ、`company/surveys`・`company/dashboard`・`user/surveys`・回答画面のルート/コントローラ/ビューを追加する構成が自然。
- ステータスや種別は Enum 化推奨（既存 `App\Enums\UserRole` / `Gender` に倣う）。
- 属性絞り込みの「部署/性別/年代」は既存の部署マスタ・`Gender` Enum・ユーザー属性を利用。
